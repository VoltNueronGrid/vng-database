1. Introduction & Overview

What is VoltNueronGrid?

VoltNueronGrid is a modern, Rust-first HTAP (Hybrid Transactional/Analytical Processing) database designed to replace in-memory MDAP (Multidimensional Analysis Platform) systems with a persistent, scalable, low-latency database engine.

Key Principles

  • ✅ Rust-first for memory safety, performance, and zero-cost abstractions
  • ✅ SOLID architecture with strict interface boundaries and plugin extensibility
  • ✅ ANSI SQL compliant baseline with documented extensions
  • ✅ Scale-up and scale-out in single unified architecture
  • ✅ No OOM or database crashes — guaranteed reliability
  • ✅ Local laptop and cloud SaaS parity — same engine everywhere
  • ✅ Observability by default — audit trails, metrics, structured logging
  • ✅ Zero data loss — ACID transactions, WAL, Raft consensus

Core Technologies

Backend Language Rust 1.75+ Memory safety, blazing speed, no GC pauses
HTTP Framework Axum (Tokio ecosystem) 70+ REST endpoints, async/await
Frontend React 18.3 + TypeScript 5.6 Modern UI with Monaco Editor
Bundler (Frontend) Vite 6.0 Lightning-fast dev server and builds
Desktop App Tauri 2.1 Lightweight cross-platform desktop client
Consensus Raft Multi-node HA with automatic failover
Durability Write-Ahead Log (WAL) + Snapshots Crash recovery and point-in-time restore

3. Backend Features

3.1 SQL Engine

  • Parser: Full ANSI SQL with extensions (window functions, CTEs, JSON operators)
  • AST: Complete abstract syntax tree representation
  • Planner: Logical query planning with cost-based optimization
  • Statement Types: SELECT, INSERT, UPDATE, DELETE, CREATE TABLE/VIEW/INDEX/FUNCTION, ALTER, DROP, TRUNCATE
  • Supported Locales: en-US, fr-FR, es-ES (extensible)
  • Functions: Aggregates, scalar, window, UDF support

3.2 HTAP (Hybrid Transactional/Analytical Processing)

Core Innovation: Single engine seamlessly handles OLTP (transactional) and OLAP (analytical) workloads with automatic intelligent routing.
  • Query Routing: Automatic detection of workload type (OLTP/OLAP/Hybrid)
  • OLTP Path: Row store with pessimistic locking, optimal for INSERT/UPDATE/DELETE
  • OLAP Path: Columnar store with SIMD vectorization, optimal for GROUP BY, JOIN, aggregates
  • Hybrid Path: Mixed workloads with adaptive strategies
  • Real-Time Sync: Sub-millisecond MVCC snapshots from row to columnar store
  • Replication Lag: Visible via metrics endpoint for SRE monitoring

3.3 MVCC (Multi-Version Concurrency Control)

  • Snapshot Isolation: Readers never block writers; readers see consistent point-in-time snapshots
  • Transaction IDs (XIDs): Globally unique transaction identifiers for version tracking
  • Write-Conflict Detection: Via XID versioning, automatic rollback on conflict
  • Isolation Levels: READ_COMMITTED (default), REPEATABLE_READ
  • Savepoint Support: Nested rollback to named savepoints within transactions
  • Visibility Rules: Tuple-level visibility checks based on XID history

3.4 Pessimistic Locking

  • Lock Types: Shared, Update (intent), Exclusive
  • Lock Scope: Row-level, key-range, partition-intent
  • Deadlock Detection: Wait-for-graph algorithm with victim selection
  • Timeout Support: Configurable per lock acquisition request
  • NOWAIT & SKIP LOCKED: Oracle-compatible modes for conditional locking
  • Metrics: Contention tracking, lock hold times, rollback reasons

3.5 RBAC (Role-Based Access Control)

Three-Tier Authentication Model:
1. Admin (VNG_ADMIN_API_KEY) → 2. Operator (x-vng-operator-id) → 3. Tenant (x-vng-tenant-id + x-vng-user-id)
  • Admin Role: Full control, dangerous operations, server management
  • Operator Role: Read queries, schema inspection, health checks
  • Tenant Role: Data-level isolation, user-scoped operations
  • Privilege Matrix: Read, Write, Admin, Execute, Transfer permissions
  • Resource Grants: Table-level, schema-level, function-level granularity
  • HTTP Status Codes: 401 (missing/invalid creds), 403 (insufficient privilege)

3.7 Data Ingestion

  • Formats: CSV (SIMD), Parquet, JSON (NDJSON + array), Excel (.xls/.xlsx)
  • Throughput: Multi-threaded chunked reading for blazing speed
  • Streaming: Pluggable connector model (FTP, S3, Azure Blob, WebDAV)
  • Schema Registry: Centralized schema management and validation
  • Batch Config: Batch size, chunk count, parallelism tuning
  • Cursor Replay: Resume interrupted ingestion from checkpoint

3.8 Distributed Caching

  • Native Cache Engine: Redis-compatible protocol support
  • Cache Types: Result cache, metadata cache, hot-key acceleration
  • TTL Policies: Per-key expiration and automatic invalidation
  • Tenant-Aware Quotas: Per-tenant cache size limits
  • Distributed Mode: Cross-node cache coherency

3.9 Audit & Compliance

  • Immutable Audit Log: Append-only tamper-evident events
  • Event Types: DDL, DML, AUTH, ADMIN, QUERY
  • Metadata: Timestamp, user, tenant, session ID, correlation ID
  • Export & Purge: Compliance-grade retention management
  • SIEM Integration: JSON export for external analysis
  • Chain Verification: Cryptographic proof of integrity
