4. Frontend Features (VoltNueronGrid Studio)

What is Studio?

A professional React-based web UI for VoltNueronGrid database management, SQL development, and monitoring.

Technology Stack

FrameworkReact 18.3.1Component-based modern UI
LanguageTypeScript 5.6Type-safe development
BundlerVite 6.0HMR, fast builds
EditorMonaco Editor 4.6VS Code-like SQL editor
State ManagementZustand 5.0Lightweight store
Desktop (Optional)Tauri 2.1Cross-platform binary
TestingPlaywright 1.59E2E test automation

Key Features

  • SQL Editor: Monaco Editor with syntax highlighting, autocomplete, error detection
  • Query Results: Tabular display, pagination, export to CSV/JSON
  • Schema Explorer: Tree view of databases, schemas, tables, columns, indexes
  • Connection Management: Save profiles, test connections, quick switch
  • Data Import Wizard: CSV, Parquet, JSON, Excel with preview
  • Data Export Wizard: Dump tables to various formats
  • DDL Viewer: View CREATE TABLE, CREATE VIEW, CREATE INDEX statements
  • Table Editor: Inline editing with row-by-row changes
  • Query History: Search, re-run, save favorite queries
  • Performance Monitor: Real-time query metrics, lock contention, cache hit rates
  • Admin Panel: User management, roles, permissions, audit logs
  • AI Chat Assistant: MCP-powered database helper

5. Running Locally (Development Setup)

5.1 Prerequisites

ToolMin VersionCheck Command
Rust1.75rustc --version
Cargo1.75cargo --version
Node.js20 LTSnode --version
npm10npm --version
Python3.11python3 --version
curlanycurl --version

5.2 Install Rust

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh source $HOME/.cargo/env

5.3 Clone & Setup

git clone https://github.com/Pavan-Pvj/polap-db.git cd polap-db cp deploy/local/vng.env.example .env # Edit .env and set VNG_ADMIN_API_KEY and other values source .env

5.4 Build Backend

# Build just the database service cargo build --release -p voltnuerongridd # Or build the entire workspace (all crates + drivers) cargo build --release

5.5 Run Backend

Minimum Configuration (HTTP Only)

./target/release/voltnuerongridd
Default ports: HTTP 8080, Native 9090 (disabled unless VNG_NATIVE_ENABLED=true)

With Native Listener Enabled

VNG_NATIVE_ENABLED=true \ VNG_NATIVE_PORT=9090 \ VNG_ADMIN_API_KEY=changeme \ ./target/release/voltnuerongridd

With TLS on Native Listener

VNG_NATIVE_ENABLED=true \ VNG_NATIVE_TLS_CERT=/path/to/server.crt \ VNG_NATIVE_TLS_KEY=/path/to/server.key \ VNG_ADMIN_API_KEY=changeme \ ./target/release/voltnuerongridd

5.6 Verify Backend

curl http://localhost:8080/health # Expected: {"status":"ok"} # SQL analyze curl -X POST http://localhost:8080/api/v1/sql/analyze \ -H "Content-Type: application/json" \ -d '{"sql":"SELECT 1"}' # Schema registry curl http://localhost:8080/api/v1/ingest/schema/registry

5.7 Build & Run Frontend

cd ui/voltnuerongrid-studio npm install npm run dev
Frontend will be available at http://localhost:1420

5.8 Configure Frontend

Edit src/config.ts or use VS Code settings:

{ "voltnuerongrid.baseUrl": "http://127.0.0.1:8080", "voltnuerongrid.transportMode": "auto", "voltnuerongrid.nativeEndpoint": "" }

5.9 Quick-Start Script

For automated setup:

bash deploy/local/install.sh

This script:

  • Checks all prerequisites
  • Copies vng.env.example to .env
  • Builds the release binary
  • Starts the server in background
  • Prints health check URL

5.10 Troubleshooting

Port 8080 already in useSet VNG_HTTP_PORT=8081
rustc not foundInstall Rust via rustup
Health check returns 503Wait 2-3 seconds and retry
Native listener not respondingSet VNG_NATIVE_ENABLED=true
npm dependencies failDelete node_modules and package-lock.json, run npm install again

7.3 Multi-Node Configuration (YAML)

cluster: mode: multi replication_factor: 3 network: http_bind: "0.0.0.0:8080" sql_bind: "0.0.0.0:5432" gossip_bind: "0.0.0.0:7000" storage: data_dir: "/var/lib/voltnuerongrid" htap: sync_mode: "near_realtime" autonomous: mode: "supervised"

7.4 Startup & Cluster Formation

cd deploy/local # Start all 3 nodes docker-compose -f multi-node.yml up -d # Wait for cluster formation (30-60 seconds) sleep 60 # Verify cluster topology curl http://localhost:18080/api/v1/admin/cluster/topology \ -H "x-vng-admin-key: admin-key"

7.5 Add/Remove Nodes Dynamically

#!/bin/bash # Add node-4 curl -X POST http://localhost:18080/api/v1/admin/cluster/nodes/manage \ -H "Content-Type: application/json" \ -H "x-vng-admin-key: admin-key" \ -d '{ "action": "add", "node_id": "node-4", "role": "follower", "desired_status": "active", "total_cpu_cores": 8, "total_ram_mb": 16384, "reason": "scale_out_for_peak_load" }' # Remove node-3 (migrate sessions to node-1) curl -X POST http://localhost:18080/api/v1/admin/cluster/nodes/manage \ -H "Content-Type: application/json" \ -H "x-vng-admin-key: admin-key" \ -d '{ "action": "remove", "node_id": "node-3", "target_node_id": "node-1", "reason": "scale_in_cost_optimization" }'

7.6 Failover Testing

# Simulate leader failure docker kill voltnuerongridd-1 # Cluster detects failure in ~5 seconds sleep 5 # Verify new leader elected curl http://localhost:28080/api/v1/admin/cluster/topology \ -H "x-vng-admin-key: admin-key" # Restore original leader docker start voltnuerongridd-1

7.7 Monitoring Multi-Node Cluster

# Check Raft status on each node curl http://localhost:18080/api/v1/cluster/raft/status curl http://localhost:28080/api/v1/cluster/raft/status curl http://localhost:38080/api/v1/cluster/raft/status # View replication lag curl http://localhost:18080/api/v1/store/htap/lag \ -H "x-vng-operator-id: operator-1" # List active transactions across cluster curl http://localhost:18080/api/v1/admin/sql/transactions/control \ -H "x-vng-admin-key: admin-key" \ -d '{"action":"list"}'
