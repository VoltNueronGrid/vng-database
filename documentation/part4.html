  • Privilege Actions: Read, Write, Admin, Execute, Transfer

  • Enforcement: Pre-execution check on every SQL statement
  • Audit: All access attempts logged to audit trail
  • Never Bypass: No skip_auth flags, no hardcoded exceptions

13.4 Encryption

In Transit

  • HTTP Listener: HTTPS with TLS 1.3 (via VNG_HTTP_TLS_CERT/KEY)
  • Native Listener: TLS 1.3 + mTLS (VNG_NATIVE_TLS_CERT/KEY + VNG_NATIVE_MTLS_CA)
  • Node-to-Node: Mutual TLS for Raft and gossip
  • Certificates: Must be signed by trusted CA in production

At Rest

  • Write-Ahead Log: Transparent encryption via KMS key envelope
  • Data Files: Optional TDE (Transparent Data Encryption)
  • Key Rotation: Online, no service interruption
  • Backup Encryption: Automatic when storing to cloud

17. Troubleshooting & FAQ

17.1 Common Issues

Port Already in Use

Error: Address already in use (os error 98) Solution: Set VNG_HTTP_PORT=8081 (or any free port)

rustc Not Found

Error: rustc: command not found Solution: curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

Health Check Returns 503

Error: Service Unavailable Cause: Server still starting up Solution: Wait 2-3 seconds and retry

Cluster Nodes Not Forming

Check VNG_RAFT_PEERS is correctly set Verify network connectivity between nodes Check firewall allows port 7000 (gossip)

Frontend Cannot Connect to Backend

Check voltnuerongrid.baseUrl setting in VS Code Verify backend is running: curl http://localhost:8080/health Check CORS headers if needed

17.2 Performance Issues

Slow Queries

1. Use tools/schema to understand table structure 2. Use POST /api/v1/sql/analyze to see plan 3. Check lock metrics: GET /api/v1/sql/pessimistic-lock/metrics 4. Look for missing indexes 5. Consider HTAP routing: POST /api/v1/sql/route

High Lock Contention

1. Check lock metrics endpoint 2. Identify blocking transactions: GET /api/v1/admin/sql/transactions/control 3. Reduce isolation level if possible 4. Use NOWAIT or SKIP LOCKED options 5. Increase VNG_LOCK_TIMEOUT_MS if transient

High Memory Usage

1. Reduce VNG_QUERY_MEMORY_LIMIT_MB 2. Decrease VNG_CONNECTION_POOL_SIZE 3. Reduce VNG_CACHE_MAX_ENTRIES 4. Check for slow queries consuming memory 5. Monitor /api/v1/sre/reliability/status

17.3 Cluster Issues

Leader Election Stuck

Check Raft status: GET /api/v1/cluster/raft/status Verify quorum (need 3+ nodes for 3-node cluster) Check network connectivity between nodes Review logs for Raft error messages

Node Replication Lag High

Check HTAP sync lag: GET /api/v1/store/htap/lag Increase htap.sync_mode from on_demand to near_realtime Check network bandwidth between nodes Look for slow disk I/O on follower nodes

17.4 Security Issues

401 Unauthorized Errors

Check auth headers are present: - x-vng-admin-key (for admin ops) - x-vng-operator-id (for operator ops) - x-vng-tenant-id + x-vng-user-id (for tenant ops)

403 Forbidden Errors

Auth present but insufficient privilege - Check role/permission for the operation - Verify tenant scoping for tenant operations - Ensure admin key is provided for admin endpoints
