1. Introduction & Overview

What is VoltNueronGrid?

VoltNueronGrid is a modern, Rust-first HTAP (Hybrid Transactional/Analytical Processing) database designed to replace in-memory MDAP systems with a persistent, scalable, low-latency database engine.

Key Principles

  • ✅ Rust-first for memory safety and performance
  • ✅ SOLID architecture with strict interface boundaries
  • ✅ ANSI SQL compliant baseline with extensions
  • ✅ Scale-up and scale-out in single architecture
  • ✅ Zero data loss — ACID transactions, WAL, Raft consensus
  • ✅ Local and cloud parity — same engine everywhere
  • ✅ Observability by default — audit trails, metrics, logging

3. Backend Features

3.1 SQL Engine

  • Parser: Full ANSI SQL with extensions
  • Statements: SELECT, INSERT, UPDATE, DELETE, DDL, DML
  • Functions: Aggregates, scalar, window, UDF support
  • Localization: en-US, fr-FR, es-ES (extensible)

3.2 HTAP (Hybrid Processing)

Smart Routing: Automatically detects workload type and routes to OLTP (row store) or OLAP (columnar) path

3.3 MVCC (Multi-Version Concurrency Control)

  • Snapshot isolation for readers
  • Readers never block writers
  • Point-in-time consistent snapshots
  • Automatic write-conflict detection

3.4 Pessimistic Locking

  • Row-level, key-range, partition locks
  • Shared, update, exclusive modes
  • Deadlock detection with victim selection
  • NOWAIT and SKIP LOCKED support

3.5 RBAC (Role-Based Access Control)

3-Tier Auth: Admin (VNG_ADMIN_API_KEY) → Operator (x-vng-operator-id) → Tenant (x-vng-tenant-id + x-vng-user-id)

7. Multi-Node Cluster Deployment

Architecture

3-Node Raft Cluster

Start Cluster

cd deploy/local docker-compose -f multi-node.yml up -d # Wait for formation sleep 60 # Verify topology curl http://localhost:18080/api/v1/admin/cluster/topology \ -H "x-vng-admin-key: admin-key"

Configuration

cluster: mode: multi replication_factor: 3 network: http_bind: "0.0.0.0:8080" gossip_bind: "0.0.0.0:7000" htap: sync_mode: "near_realtime"

curl -X POST http://localhost:18080/api/v1/admin/cluster/nodes/manage \ -H "Content-Type: application/json" \ -H "x-vng-admin-key: admin-key" \ -d '{ "action": "add", "node_id": "node-4", "role": "follower" }'

17. Troubleshooting & FAQ

Common Issues

Port Already in Use

Error: Address already in use Solution: Set VNG_HTTP_PORT=8081

Rust Not Found

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

Health Check Returns 503

Cause: Server still starting Solution: Wait 2-3 seconds and retry

Cluster Nodes Not Forming

Check: VNG_RAFT_PEERS is set correctly Check: Network connectivity on port 7000 Check: Firewall allows gossip protocol

Frontend Can't Connect

Check: voltnuerongrid.baseUrl in VS Code settings Verify: Backend running (curl http://localhost:8080/health) Check: CORS headers if needed

Performance Issues

Slow Queries

  • Use tools/schema to understand structure
  • Use POST /api/v1/sql/analyze for query plan
  • Check lock metrics
  • Look for missing indexes
  • Check HTAP routing

High Lock Contention

  • Check lock metrics endpoint
  • Identify blocking transactions
  • Reduce isolation level if possible
  • Use NOWAIT/SKIP LOCKED
